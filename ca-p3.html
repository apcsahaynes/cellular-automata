Cellular Automata

Page 3 — Images as 2D Grids

On this page, you'll manipulate pixels in a photograph. An image is a rectangular grid of pixels — each one at a specific row and column — and working with it introduces the same structure you'll use for 2D arrays on the next page.

A digital image is a rectangular grid of pixels. Each pixel sits at a column (x) and a row (y), and holds a color. In Processing, images are objects created from the PImage class. They don't have a traditional constructor. You load and display one like this:

PImage img; //creates an image object

void setup() {
    size(800, 834); // size of image
    img = loadImage("pillars.jpg"); //sets the image to a file in your sketch folder
    image(img, 0, 0);  // draw starting at the top-left corner
}

Open a new sketch and download the Hubble "Pillars of Creation" Save it as pillars.jpg and drag it into the the Processing app.

Copy the code and run it. You should see the image. Note: you may get an warning about pixel density. You can ignore it.

img.get(x, y) returns the color at column x, row y. img.set(x, y, c) writes a color there. Colors are stored as integers; red(c), green(c), and blue(c) extract each channel as a value from 0 to 255. color(r, g, b) builds a new color.

To process every pixel, you'll need to iterate over every row and column. The image has img.height rows and img.width columns.

The code will be similar to what you did in Breakout: nested for loops. The outer loop steps through rows from top to bottom. The inner loop moves across each row one column at a time.

The filters above treat each pixel independently; neighboring pixels are not dependent on each other. A blur is different: each pixel's new color is an average of itself and its neighbors. This creates the same problem as the cellular automata. Uou can't read and write at the same time. Reading a pixel after it's been updated gives you the new value, not the original.

The fix is to read from one image and write to a separate copy.

PImage img;
PImage img;
PImage blurred;

void setup() {
    size(800, 834);
    img = loadImage("pillars.jpg");
    blurred = createImage(img.width, img.height, RGB); //creates an empty image
}

void draw(){
  image(img, 0, 0);
}

void keyPressed() {
    blur(img, blurred);  // compute next state
    img = blurred.copy(); // make blurred the new current image
}
6.

Update your sketch to match the above code. Complete blur: Loop over every pixel, skipping the outer border. For each pixel, average it with its four direct neighbors (above, below, left, right) and write the result to destination. Average each channel separately.

Hint: why skip the border?
Similar to the cellular automata, if we don't ignore the edges, we'll access indexes that are out of bounds. The same idea applies, but we need to skip the edges in both dimensions.
Hint: averaging the channels

Again, similar to the cellular automata, we want to access the neighbors in relation to the current pixel's (x, y) location. We. You'll need to get and store five colors: center, above, below, left, right. Then:

On the Exam

The standard pattern for visiting every element of a 2D structure:

for (int r = 0; r < rows; r++) {
    for (int c = 0; c < cols; c++) {
        // use element at row r, column c
    }
}

The outer loop controls which row. The inner loop moves across that row one column at a time. For images, rows are y values (top to bottom) and columns are x values (left to right).
