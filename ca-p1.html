Cellular Automata

Page 1 — Arrays and Rules

In this lab, you'll use arrays to simulate a cellular automaton: a grid of cells that change state over time based on simple rules determined by the states of their immediate neighbors.

On this page, you'll get comfortable with arrays and understand how a rule works by applying it by hand.

An array is a fixed-size sequence of values. Unlike ArrayList, arrays cannot change their size once they have been created. In Java, you declare one like this:

int[] cells = {0, 0, 0, 1, 0, 0, 0};
println(cells[3]);     // → 1
println(cells.length); // → 7

Positions start at 0. The last valid position is cells.length - 1, which here is 6. Accessing cells[7] throws an ArrayIndexOutOfBoundsException.

A cellular automaton is a set of cells, each either alive (1) or dead (0). At each step, every cell produces a new value based on itself and its two immediate neighbors. The cell being updated is called the center. The center together with its left and right neighbors are called the cell's neighborhood.

Since each of the three cells in a neighborhood is 0 or 1, there are exactly 8 possible neighborhoods. A rule is a table that says: given a neighborhood, what should the center cell's next value be?

There are many rules. Here is Rule 90. Each row of the table is one possible neighborhood, and the last column says what the center cell becomes in the next generation:
