Page 4 — 2D Arrays

On this page, you'll move from pixels to integers. A 2D array of int values has the same row-and-column structure as the image from page 3, with simpler syntax for reading and writing values.

On the previous page, you read and wrote pixels with img.get(x, y) and img.set(x, y, c). A 2D integer array works the same way conceptually, but the access syntax is more direct. Instead of one dimension, you give it two indexes.

int[][] cells = new int[6][8];  // 6 rows, 8 columns, all zeros
cells[3][2] = 1;                // row 3, column 2
println(cells[3][2]);           // → 1
println(cells.length);          // → 6  (number of rows)
println(cells[0].length);       // → 8  (number of columns)

On the Exam

The standard traversal pattern for a 2D array:

for (int r = 0; r < cells.length; r++) {
    for (int c = 0; c < cells[r].length; c++) {
        // use cells[r][c]
    }
}

cells.length is the number of rows. cells[0].length is the number of columns.

In 2D, a cell can have up to 8 neighbors. You reach each neighbors by adding an offset to the row and/or column. The cardinal neighbors (up,down,left,right) are offset by a single row or column. Diaganol neighbors offset both the row and column.

You can avoid out of bounds errors entirely by looping from row 1 to second last row. This is the same trick as skipping the edges in the 1D automaton and blur effect, and it means you never try to access a row or column that doesn't exist.

Self-check

1. What does this code print?

int[][] grid = new int[4][4];
grid[1][2] = 1;
grid[2][1] = 1;
grid[2][3] = 1;

int total = 0;
for (int r = 1; r < grid.length - 1; r++) {
    for (int c = 1; c < grid[0].length - 1; c++) {
        if (grid[r][c] == 1) {
            total += grid[r-1][c] + grid[r+1][c] +
                     grid[r][c-1] + grid[r][c+1];
        }
    }
}
println(total);

Choose one:

A) 2
Correct. Only grid[2][1] and grid[2][3] are 1 and in the interior loop range. For grid[2][1]: its four cardinal neighbors are grid[1][1]=0, grid[3][1]=0, grid[2][0]=0, grid[2][2]=0 — total 0. For grid[2][3]: neighbors are grid[1][3]=0, grid[3][3]=0, grid[2][2]=0, grid[2][4] — wait, column 4 is out of the interior loop range, so grid[2][3] is not visited. Only grid[1][2] and grid[2][1] are visited. grid[1][2]'s neighbors sum to 0+0+0+0=0 (none of its cardinal neighbors are 1). grid[2][1]'s neighbors: above is grid[1][1]=0, below is grid[3][1]=0, left is grid[2][0]=0, right is grid[2][2]=0 — 0. So... hmm. Let me recheck: grid[1][2]=1, its neighbors: grid[0][2]=0, grid[2][2]=0, grid[1][1]=0, grid[1][3]=0 — sum 0. grid[2][1]=1: grid[1][1]=0, grid[3][1]=0, grid[2][0]=0, grid[2][2]=0 — sum 0. Total is 0. My answer key was wrong — correcting below.
B) 0
Correct. The loop visits interior cells (rows 1-2, columns 1-2). Of those, grid[1][2] and grid[2][1] are 1. For each, the code sums its four cardinal neighbors. None of those neighbors are 1, so total stays 0.
C) 3
That is the total number of live cells, not what the loop computes. The loop only sums neighbors of live interior cells.
D) This throws an ArrayIndexOutOfBoundsException.
No out-of-bounds access occurs. The loop runs from r=1 to r<grid.length-1 and c=1 to c<grid[0].length-1, so all neighbor accesses (r±1, c±1) are safe.

2. A student modifies the loop to start at r = 0 instead of r = 1. What happens?

Choose one:

A) The result is the same, because row 0 has no live cells.
The result being the same in this specific case doesn't mean the code is correct. When r=0, the line grid[r-1][c] tries to access grid[-1][c].
B) An ArrayIndexOutOfBoundsException is thrown when r=0, because grid[r-1][c] accesses row -1.
Correct. Even if row 0 happens to have no live cells in this example, the loop still evaluates grid[r-1][c] for every cell — and row -1 does not exist.
C) The loop skips row 0 automatically.
Java does not skip rows automatically. If the loop starts at r=0, it processes row 0.
D) The total increases by the values in row 0.
The code would crash before producing any output.
